workers.default.ojs.ojs_api.generate_frontmatter(article_id)[source]¶

Generate frontmatter page for articles in OJS via Frontmatter-Plugin.

Does a simple GET request to the OJS plugin passing on IDs of documents already present in OJSself.

Authorization is done with a specific test user.

Parameters:article_id – ID of the article in OJS
Returns:tuple containing the response code and text
workers.default.ojs.ojs_api.publish(import_xml_file_path, journalcode)[source]¶

Publish the documents referenced in the passed XML via OJS-Import-Plugin.

The paramter file contains OJS-specific XML which contains file paths to documents which shall be imported to OJS. It is read and passed to a POST request (in UTF-8) to the OJS import plugin.

Server address and port can be given optionally.

The Import-Plugin needs authentication which is (for now) hard-coded.

Parameters:
  • import_xml_file_path (str) – OJS-Import-XML
  • journalcode (str) – Name of the journal that will be imported to
Returns:

Tuple of return code and text of the POST request to OJS
